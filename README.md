# 学生信息管理系统

基于命令行的学生信息管理系统，使用 C99 开发。支持增删改查、多字段排序、多维统计，哈希表 O(1) 查找，数据持久化到文本文件。

## 环境要求

- 操作系统：Linux / macOS / Windows（Git Bash / WSL）
- 编译器：GCC、Clang 或 CMake

## 编译

```bash
# 方式一：编译脚本（自动检测 CMake > GCC > Clang）
./compile.sh              # 编译
./compile.sh clean        # 清理编译产物

# 方式二：CMake
mkdir build && cd build && cmake .. && make

# 方式三：手动编译
gcc -Wall -Wextra -std=c99 -O2 -o studentms src/*.c
```

产物为可执行文件 `studentms`。

## 运行

```bash
./studentms
```

首次运行自动从 `students.txt` 加载数据（文件不存在则静默创建空数据库）。

## 功能菜单

```
+========================================+
|         学生信息管理系统 v1.0           |
+========================================+
|  1. 添加学生信息                        |
|  2. 显示所有学生信息                    |
|  3. 按学号查找学生                      |
|  4. 按学号删除学生                      |
|  5. 按学号修改学生信息                  |
|  6. 排序（学号/姓名/年龄/成绩/学院）    |
```

### 操作说明

| 选项 | 功能 | 说明 |
|------|------|------|
| 1 | 添加 | 依次输入学号、姓名、性别、年龄、成绩、学院（米哈游游戏名） |
| 2 | 显示 | 表格形式列出所有学生，CJK 感知列对齐 |
| 3 | 查找 | 输入学号，哈希表 O(1) 即时查找 |
| 4 | 删除 | 输入学号删除记录 |
| 5 | 修改 | 输入学号后逐字段修改，回车保留原值 |
| 6 | 排序 | 6 种算法 × 5 字段（含学院）× 升降序，计时输出 |
| 7 | 统计 | 男女比例、成绩、年龄分布、学院分布 |
| 8 | 保存 | 手动将内存修改写入文件 |
| 0 | 退出 | 有未保存修改时询问是否保存 |

### 输入规则

| 字段 | 范围 | 备注 |
|------|------|------|
| 学号 | 1 ~ 1999999999 | 唯一，不可重复 |
| 姓名 | 最多 31 字节 | 支持中文 |
| 性别 | "男" 或 "女" | 严格校验 |
| 年龄 | 0 ~ 150 | 整数 |
| 成绩 | 0.00 ~ 100.00 | 浮点数 |
| 学院 | 最多 31 字节 | 米哈游游戏名（原神/星穹铁道/崩坏3/绝区零 等） |

输入超长或非法字符时自动拒绝，不会导致崩溃。

### 数据文件

数据存储在 `students.txt`，每行 6 个字段，空格分隔：

```
学号 姓名 性别 年龄 成绩 学院
1000000217 芙宁娜 女 17 10.90 原神
1000033218 姬子 女 28 3.60 星穹铁道
```

可手动编辑该文件，不合规行（字段数不对、数值越界、学号重复）会被自动跳过并在启动时输出警告。

## 架构设计

### 模块划分

| 文件 | 职责 |
|------|------|
| `src/main.c` | 程序入口、主循环、菜单调度 |
| `src/menu.c` | 菜单界面打印 |
| `src/student.c` | 链表操作（增删改查）、统计、脏标记 |
| `src/sort.c` | 6 种排序算法（冒泡/选择/插入/归并/快速/梳排序） |
| `src/input.c` | 安全输入（防缓冲区溢出、类型校验、范围限制） |
| `src/fileio.c` | 文件加载（逐行校验）、原子保存 |
| `src/display.c` | CJK 感知的终端列宽计算与对齐打印 |
| `src/hash.c` | 链地址哈希表（动态扩容，O(1) ID 查找） |

### 数据结构

```c
typedef struct Student {
    int    id;
    char   name[32];
    char   gender[8];
    int    age;
    float  score;
    char   college[32];       // 学院（游戏名）
    struct Student *next;     // 链表后继
    struct Student *hash_next;// 哈希冲突链
} Student;
```

- **单向链表**：用于遍历显示、排序、释放
- **哈希表**：初始 1024 槽，负载因子 > 0.75 自动扩容 2 倍，冲突链地址法

### 性能特征

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 按 ID 查找 | O(1) | 哈希表直接定位 |
| 添加（唯一性校验）| O(1) | 哈希表判重 |
| 修改（定位）| O(1) | 哈希表查找 |
| 文件加载 | O(n) | 逐行解析 + 哈希表去重 |
| 删除 | O(n) | 需扫描前驱结点摘除 |
| 排序 | O(n log n) | 归并/快速排序 |
| 显示/统计 | O(n) | 遍历链表 |

### 保存策略

- 增删改操作**不立即写文件**，仅标记内存脏标志
- 退出时若有未保存修改，提示 `是否保存后退出? (y/n)`，用户可自由选择
- 手动保存（选项 8）可随时持久化
- 文件写入采用**原子保存**：临时文件 → 重命名，失败自动回滚
